# 穴埋め暗記演習用軽量ウェブアプリ

**URLを開いて0秒で演習開始**。 「準備の手間」を削減した、汎用的で軽量な暗記演習（ドリル）アプリケーションです。

HTMLファイル内の設定（JSON）を差し替えるだけで、英単語・化合物名・歴史年号など、様々なドメインの学習アプリを生成可能です。<br>
※デフォルトでは英語の演習問題になっています。

## メリット

学習者の目線で見れば、多くの高機能な暗記学習アプリがありますが、「アプリのインストール」「データベースの作成」「教材のインポート」など、学習開始までに**高いモチベーションと環境を準備するための時間**、場合によっては**ITスキル**を要求します。
本ツールは、教育者側の目線に立ち、**学習者がワンクリックで演習に参加できる環境**を提供するために設計されました。

### 1. 0秒で演習開始
* **インストールなし・ログインなし:** 学習者に必要なのはブラウザだけ。URLを開いた瞬間に演習が始まります。
* **環境構築不要:** 学習者側での細かな設定は不要。PC、スマホ、タブレットで動作します。

### 2. 紙教材の制約からの解放
* **印刷コスト・待ち時間ゼロ:** 教員が演習問題を印刷するために「印刷室へ行く」「プリンターの順番待ちをする」「紙詰まりを直す」……そんな時間は不要です。
* **「教材忘れ」の撲滅:** スマホさえあれば、そこが演習の場所です。「プリントを家に忘れました」という言い訳を不可能にします。

### 3. 純粋な学習環境
* **広告なし・課金なし:** 集中力を阻害するバナー広告や、機能制限はありません。
* **カスタマイズ性:** 画面デザインはCSSでカスタマイズできます。学習者の特性に合わせて、文字サイズや配色を編集できます。

## 技術的な特徴

### 1. オフライン動作設計
* **依存関係の少なさ:** 純粋なHTML/JS/CSSで動作。
* **ポータビリティ:** ファイル一式をダウンロードすれば、インターネット接続のないオフライン環境（機内など）でも動作。

### 2. 学習効率を上げるための出題ロジック
* **未習得問題の優先:** `LocalStorage` を使用して学習履歴をブラウザ内に保存。過去に出題されていない問題を優先的に出題。

### 3. 複数の正答に対応
* **柔軟な正誤判定:** 英語での動詞の活用における`learned/learnt` や `burned/burnt` のような、複数の正答に対応。
* **簡単な実装:** 解答データを `/` で区切るだけのシンプルなデータ構造で、複数正答に対応した正誤判定を実施。

### 4. 科目（ドメイン）に依存しない汎用設計
* **データとロジックの分離:** アプリケーションロジック（`app.js`）と学習データ（HTML内のJSON Config）を分離。
* **拡張性:** 英単語に限らず、`config` オブジェクトを書き換えるだけで、他の暗記用ドリルとしても転用可能。


## 開発背景

以下の3点を意識し、手軽な学習環境を提供するために開発。

1.  **学習機会の保障**
    * ネットワーク接続の有無に依存せず、多くの学習者に等しく反復練習の場を提供する。
2.  **ペーパーレス演習の実現**
    * 物理的な教材を持ち歩くことなく、スマホやPCがあればいつでもどこでも演習可能にする。
3.  **教材開発コストの低減**
    * 共通のフレームワークで多用途（英語、理科、社会など）に展開可能にし、演習問題作成にかかる負荷を下げる。


## 導入・使用方法

インターネット接続なしで動作させる場合や、問題をカスタマイズしたい場合は、以下のようにしてください。

1. ファイルを一式ダウンロードしてください。
2. HTMLファイルをブラウザで開くと動作します。
3. 問題データを変更したい場合は、HTMLファイル内の `const quizData = { ... }` 部分を編集してください（生成AIを用いたガイドは後述）。
4. HTMLファイルを公開したい場合は、適切な場所で公開してください。

※正解判定時、大文字小文字は無視して判定されます。


### フォルダ（ディレクトリ）構成の例
ダウンロードしたファイルは、以下のように同じフォルダ（ディレクトリ）に入れてください。

```
📁 クイズアプリフォルダ
 ├─ 📄 HTMLファイル群   （verbs.html や 自作のHTML）
 ├─ 📄 app.js      （ロジック本体）
 └─ 📄 style.css   （デザイン）
```

## 自作問題への差し替えガイド（生成AI活用編）

このアプリは、HTMLファイルの中にあるデータを書き換えるだけで、「英単語」だけでなく「理科の元素記号」「歴史の年号」「漢字の読み」など、様々な暗記ドリルに変身させることができます。

**Excel（またはスプレッドシート）** と **生成AI（ChatGPT, Google Gemini など）** があれば、オリジナルのアプリが作れるはずです。

### 必要なもの

1.  **元となるHTMLファイル**（`verbs.html` など）
2.  **新しい問題データ**（ExcelやGoogleスプレッドシートで作った表）
3.  **生成AI**（ChatGPT, Google Geminiなど、ファイルを添付できるものがおすすめ）

### 手順

#### STEP 1: 問題データを用意する

Excelなどで、問題にしたい表を作ります。

  * **1列目:** 見出し（問題のラベル。例：「日本語の意味」）
  * **2列目以降:** クイズにしたい内容（例：「英単語」「過去形」など）
  * **注意:** 正解が複数ある場合は `learned/learnt` のように半角スラッシュ `/` で区切って書いておきます。

**（例：元素記号クイズを作りたい場合）**
| 元素名 | 記号 | 英語名 |
| :--- | :--- | :--- |
| 水素 | H | Hydrogen |
| ヘリウム | He | Helium |
| リチウム | Li | Lithium |


#### STEP 2: 生成AIに「書き換え」を依頼する

生成AIのチャット画面を開き、**「元のHTMLファイル」をアップロード**した上で、以下のプロンプト（命令文）と、Excelからコピーしたデータを貼り付けて送信します。

##### 📋 コピー用プロンプト（命令文）

```text
添付したHTMLファイル（verbs.html）を元に、新しいクイズアプリを作りたいと思います。
HTML内の <script> タグにある `quizData` の内容を、以下のデータに書き換えたコードを作成してください。

【要件】
1. HTML全体ではなく、書き換える必要がある `<script>` タグの中身だけを出力してください。
2. 変数 `quizData` の中身を以下のように変更してください。
   - id: "my_quiz_01" （一意なIDにしてください）
   - title: "（ここに入れたいタイトルを書く、例：元素記号マスター）"
   - allColHeaders: （データの2列目以降のヘッダー名を使ってください）
   - rowHeaders: （データの1列目を使ってください）
   - allData: （データの2列目以降を使ってください）
3. 正解が複数ある場合（ / 区切り）は、そのまま文字列として保持してください。

【新しいデータ】
（ここにExcelの表をそのままコピペしてください）
```

#### STEP 3: コードを貼り付ける

1.  AIが生成したコード（`<script>...` の部分）をコピーします。
2.  テキストエディタ（メモ帳やVS Codeなど）で `verbs.html` を開きます。
3.  ページ下部の `<script>` ～ `</script>` の間で、設定データ（`const quizData = ...`）が書かれている部分を、生成AIが出力したコードに置き換えます。
4.  **「名前を付けて保存」** で、わかりやすい名前（例：`science.html`）にして保存します。

#### STEP 4: 完成！

保存したHTMLファイルをブラウザ（ChromeやEdge）で開いてみてください。
あなたが作ったデータでクイズが始まります！<br>
※app.jsとstyle.cssはHTMLファイルと同じフォルダ（ディレクトリ）に配置してください。

## うまくいかない時のヒント（生成AIへの追加指示）

生成AIが作ったデータでレイアウトが崩れた場合や、不要な列があった場合は、以下のように追加で指示を出してみてください。

  * **特定の列を除外したい時:**
    > 「Excelのデータの3列目はクイズに使わないので、データから除外してください」
  * **特定の列を穴埋め対象外にしたい時:**
    > 「1列目（元素記号）は常に表示して、2列目（英語名）だけを穴埋め問題にしたいです。`noQuizColumns` の設定を調整してください」
  * **項目数が多すぎる時:**
    > 「項目数が多すぎるので、`defaultRowCount` を 10 に設定し、行数制限機能を有効（`enableRowSelection: true`）にしてください」
